# 学生管理系统 — 业务错误码说明（Business Code）

## 一、说明

本业务错误码文档用于统一定义学生管理系统中各类**业务层错误状态**，用于补充 HTTP 状态码无法精确表达的业务失败原因。  

HTTP 状态码用于表示请求在协议层面的成功或失败，而业务错误码（Business Code）用于表示**具体业务规则不满足的原因**，便于前后端协作与问题定位。

---

## 二、设计原则

1. 业务错误码为五位数字  
2. 按功能模块划分区间，便于维护和扩展  
3. 同一错误码在系统中含义唯一  
4. 前端可根据业务错误码进行精细化提示  

---

## 三、错误码区间划分

| 区间 | 模块说明 |
|------|----------|
| 10000–10999 | 用户与账号相关 |
| 20000–20999 | 班级本体相关 |
| 30000–30999 | 班级成员关系 |
| 40000–40999 | 权限与认证 |
| 90000–99999 | 系统通用错误 |

---

## 四、用户与账号相关（100xx）

| 错误码 | 含义 |
|------|------|
| 10001 | 用户名已存在 |
| 10002 | 用户不存在 |
| 10003 | 密码错误 |
| 10004 | 账号已被禁用 |
| 10005 | 原密码错误 |
| 10006 | 新密码不符合规则 |
| 10007 | 验证码错误或已过期 |

---

## 五、班级相关（200xx）

| 错误码 | 含义 |
|------|------|
| 20001 | 班级人数已满 |
| 20002 | 班级不存在 |
| 20003 | 班级已被删除 |
| 20004 | 班级名称已存在 |
| 20005 | 非法的班级类型 |

---

## 六、学生与班级关系（300xx）

| 错误码 | 含义 |
|------|------|
| 30001 | 学生已在该班级中 |
| 30002 | 学生不在该班级中 |
| 30003 | 学生未分配专业班级 |
| 30004 | 学生不属于当前操作范围 |
| 30005 | 学生无资格加入该班级 |

---

## 七、权限与认证（400xx）

| 错误码 | 含义 |
|------|------|
| 40001 | 无权限执行该操作 |
| 40002 | 登录状态已失效 |
| 40003 | 角色不匹配 |
| 40004 | 非法访问受限资源 |

---

## 八、系统通用错误（900xx）

| 错误码 | 含义 |
|------|------|
| 90001 | 参数校验失败 |
| 90002 | 系统内部错误 |
| 90003 | 服务暂不可用 |

---

## 九、使用示例（说明）

```json
{
  "code": 20001,
  "message": "班级人数已满",
  "data": null
}
```

---

## 十、说明

本业务错误码文档为系统设计层规范，后续可根据实际业务扩展新的错误码区间，但需保证原有错误码语义不发生改变。
