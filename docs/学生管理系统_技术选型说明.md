# 学生管理系统 — 技术选型说明

## 一、技术选型目标

本技术选型文档用于说明学生管理系统在开发过程中所采用的主要技术方案及其选型理由，确保系统在**可实现性、可维护性、学习成本与工程规范性**之间取得平衡。

本项目定位为课程设计 / 综合实践项目，因此在技术选型上遵循以下原则：

- 技术成熟、社区活跃
- 易于学习与调试
- 能清晰体现工程思想
- 避免过度设计与不必要复杂度

---

## 二、总体架构选型

### 2.1 架构模式

- 采用 **前后端分离架构**
- 前端负责页面展示与用户交互
- 后端负责业务逻辑、权限控制与数据处理

该架构模式具有以下优势：
- 职责划分清晰，便于模块化开发
- 前后端可独立开发、独立部署
- 更贴近真实企业级项目结构

---

## 三、后端技术选型

### 3.1 Node.js

- 版本：Node.js 22.x
- 作为后端运行环境

**选型原因：**
- 基于 JavaScript，语言统一，降低学习成本
- 非阻塞 I/O 模型，适合 Web 应用开发
- 生态成熟，第三方库丰富
- 启动快，开发效率高

---

### 3.2 Web 框架（Express / Koa）

- 使用轻量级 Web 框架构建 HTTP 服务

**选型原因：**
- 框架简单直观，便于理解请求处理流程
- 不强制复杂约定，适合教学与实践项目
- 易于实现中间件机制（如登录校验、权限校验）

---

### 3.3 权限控制设计（RBAC 思想）

- 采用基于角色的访问控制（RBAC）设计思想
- 系统内置角色：学生、教师、辅导员、超级管理员

**说明：**
- 权限校验统一在服务端完成
- 本项目重点体现权限设计思想，具体实现保持简化

---

### 3.4 数据访问层

- 通过 Node.js 访问 MySQL 数据库
- 使用 ORM 或 SQL 封装方式进行数据操作

**选型原则：**
- 保持数据操作逻辑清晰
- 避免 SQL 逻辑与业务逻辑混杂

---

## 四、前端技术选型

### 4.1 前端框架

- 使用现代前端框架（如 Vue / React）构建用户界面

**选型原因：**
- 组件化开发，利于页面复用
- 数据驱动视图，适合复杂表单与列表管理
- 与前后端分离架构天然契合

---

### 4.2 前端功能设计关注点

- 登录 / 注册 / 忘记密码流程
- 根据用户角色动态展示菜单与页面
- 表单校验与用户提示
- 基础页面权限控制（展示层）

---

## 五、数据库技术选型

### 5.1 MySQL

- 版本：MySQL 8.x
- 作为系统关系型数据库

**选型原因：**
- 成熟稳定，广泛应用于 Web 系统
- 支持复杂关系建模
- 易于管理与可视化操作（Navicat）

---

## 六、开发与构建工具

### 6.1 包管理与构建工具

- npm：用于前端与后端依赖管理

### 6.2 开发工具

- IDE：VS Code / IntelliJ IDEA
- 数据库管理工具：Navicat

---

## 七、安全与规范设计说明

- 登录状态校验统一在后端完成
- 敏感操作必须通过权限校验（设计层面支持，暂不完全实现）
- 密码采用加密存储方式（不明文保存）

---

## 八、技术选型总结说明

本项目技术选型以**工程实践与学习价值**为导向，在保证系统功能完整的前提下，避免复杂框架与过度封装，适合作为学生管理系统的实践项目与展示项目。

该技术方案能够支撑当前需求，并为后续功能扩展与技术升级提供良好基础。

- 本项目未引入微服务架构
- 未引入分布式权限中心
- 未实现高并发、高可用及容灾设计
- 未涉及容器化部署与自动化运维
- 技术选型以教学实践与工程认知为主要目标
